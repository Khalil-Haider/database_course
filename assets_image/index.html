Primary Key

A primary key is one of the candidate keys chosen to uniquely identify tuples in a relation.

  • The value of primary key can never be NULL
  • The value must always be unique (no duplicates)
  • The values cannot be changed (no updates)
  • The value must be assigned when inserting a record
  • A relation is allowed to have only one primary key

Example: In the Employee table, Emp_Id is the primary key

Super Key

A super key is a combination of all possible attributes that can uniquely identify rows (or tuples) in the given relation.

  • Super key is a superset of a candidate key
  • A table can have many super keys
  • May have additional attributes not needed for unique identity

Examples in Employee Table:

{Emp_Id}, {Aadhar_No}, {Email_Id}, {Emp_Id, Aadhar_No}, {Aadhar_No, Email_Id}, {Emp_Id, Email_Id}, {Emp_Id, Aadhar_No, Email_Id}, {Emp_Id, Name}, {Emp_Id, Name, Dept_Id}

Candidate Key

A candidate key is an attribute or set of attributes which can uniquely identify a tuple.

  • A minimal super key (no redundant attributes)
  • Selected as the minimum attributes required for uniqueness
  • Not allowed to have NULL values
  • The set from which primary key can be selected

Examples in Employee Table:

{Emp_Id}, {Aadhar_No}, {Email_Id}

Alternate Key

Out of all candidate keys, only one gets selected as primary key, remaining keys are known as alternate keys.

  • Has same uniqueness properties as primary key
  • Can be used as secondary identifier
  • Often used for alternate access paths to data

Examples in Employee Table:

Since Emp_Id is the primary key, {Aadhar_No} and {Email_Id} are alternate keys

Foreign Key

A key used to link two tables together. An attribute in one table that refers to the Primary Key in another table.

  • References primary key of another table
  • Can take only values present in referenced primary key or NULL
  • May have a different name than the referenced key
  • Can take NULL value (unless constrained)
  • No restriction on uniqueness (often not unique)

Example: In Employee Table, Dept_Id is a foreign key referencing the Department table

Composite Key

A key that has more than one attribute. Also known as compound key.

  • Multiple columns together form one key
  • Used when a single attribute can't uniquely identify records
  • Common in many-to-many relationships

Example: In the customer order example table, {Cust_Id, Product_Code} forms a composite key
